Git

2021-04-28
修改上一次提交信息

git commit --amend -m <new_message>

修改历史提交信息

git rebase -i <commitId> （commitId，HEAD] 范围，将需要修改的提交 pick 改为 reward，保存即可

CRLF 报错

git config --global core.autocrlf false

远程分支删除后，本地依然能够看到该分支

git remote prune origin origin 为远程仓库名称

比较分支差异
  1. 查看dev分支存在而master分支不存在的提交git log dev ^master
  2. 查看master比dev多提交了哪些git log dev..master
  3. 比较dev与master分支的差异git log dev...master
  4. 显示所有差异文件的详细git diff dev master
  5. 比较两个分支具体文件的差异git diff dev master <具体文件路径>
  6. 显示两分支差异部分git diff dev master --stat
  7. 上述情况，显示每个提交所在的分支,例如git log -left-right dev...master
重命名本地分支与远程分支
# 重命名本地分支
git branch -m oldBranch newBranch (or git branch --move oldBranch newBranch)
# 删除远程旧分支
git push origin :oldBranch
# 推送本地新分支
git push origin newBranch
同一命名空间下无法派生同名项目（以旧项目某一分支派生新项目）
# 从旧项目克隆项目到本地
git clone <旧项目地址> -b <目标分支Branch1>
# 检查远程项目关联
git remote -v 
# 移除远程项目关联
git remote remove <origin 远程仓库名称，根据上一步填写>
# 重命名分支名称
git branch -m <Branch1> master
# 远程仓库创建新项目，记录地址URL2
# 关联当前项目与新创建的远程仓库
git remote add origin <URL2>
# 建立本地分支与远程分支的关联
git push --set-upstream origin master
# 推送本地项目至远程仓库
git push
忽略某个文件的提交
  1. 保留在版本库当中
# 忽略指定文件变更
git update-index --assume-unchanged <文件全路径>
# 取消上述命令
git update-index --no-assume-unchanged <文件全路径>
  1. 从版本库中删除
git rm -r --cached 
#or
git rm --cached <文件全路径>
回退版本
#本地回退
git reset --hard <commitId>
#推送到远程仓库 origin 远程仓库名 master 远程仓库分支
git push -u origin master -f
